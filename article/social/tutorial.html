Konsep Dasar: Struktur Virtual

Firebase Cloud Storage adalah layanan penyimpanan objek yang kuat. Hal terpenting untuk dipahami adalah: Firebase Storage tidak memiliki struktur "folder" fisik. Ia menggunakan struktur flat (datar).

Namun, jika Anda menyertakan garis miring (/) pada nama file, Firebase secara virtual akan menampilkannya seolah-olah itu adalah sebuah folder. Lihat ilustrasi di bawah:

Yang Anda Lihat (Visual/Folder)

dokumen
laporan.pdf
proposal.docx

Penyimpanan Asli (Flat Object)

"dokumen/laporan.pdf"
"dokumen/proposal.docx"

Persiapan Awal: Sebelum melakukan operasi di bawah, pastikan Anda telah menginisialisasi getStorage(app) menggunakan Firebase config Anda.

Kesimpulan & Aturan Penting

Folder itu Ilusi

Struktur foldernya adalah virtual berdasarkan string path (/). Untuk menghapus sebuah folder, Anda harus menghapus semua file di dalamnya.

Tidak Ada Edit Isi / Rename

Tidak ada operasi edit isi file atau rename bawaan. Anda harus timpa (overwrite) file lama dengan yang baru, atau download -> upload dengan nama baru -> hapus yang lama.

Keamanan Itu Wajib

Pastikan Anda selalu mengonfigurasi Firebase Security Rules agar aset dan file Anda aman dari akses publik yang tidak diinginkan.
